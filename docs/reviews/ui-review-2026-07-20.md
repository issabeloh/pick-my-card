# 網站整體 UI 檢視報告（2026-07-20）

檢視方式：Playwright 實跑全站（訪客首訪 → landing → 進工具 → 搜尋「蝦皮」→ 查無商家 → 側選單），
桌機 1440×900 與手機 390×844 兩種視口，並對照 `styles.css`／`index.html`／`js/` 程式碼證據。
定位：以「行動裝置為主、決策輔助型比較工具」的類別標準，找出違反經時間驗證之設計原則的地方。
本報告只記錄發現與調整方向，未改任何程式碼。

## 總評

底子不差：viewport 沒鎖縮放、輪播有暫停機制、有 `prefers-reduced-motion` 處理、FAQ/promos 頁乾淨、
結果有「最優回饋」徽章與法遵警語。但有幾處踩到成熟的反模式，其中兩項直接傷到比較工具的核心價值
（快速、可信的比較）。

## 發現（依嚴重度排序）

### 1. 查無商家時的「沉默 fallback」——違反系統狀態可見性（Nielsen #1）【高】

搜尋不存在的商家時，頁面不說「找不到」，直接列出各卡「一般消費回饋率」排名；
唯一線索是卡片內一行小字，搜尋詞也沒被回顯。用戶會誤以為該回饋率是指定通路回饋，
做出錯誤刷卡決策——錯誤的信任比沒有答案更糟。

**方向**：結果區頂部加狀態列——「找不到『XX』的指定通路回饋，以下為一般消費回饋排名」，附相近商家建議。

### 2. 決策關鍵資訊用全站最低可讀性樣式——對比度＋CJK 斜體反模式【高】

`.cashback-condition`（styles.css 約 4075 行，行號會漂移、以 Grep 為準）＝ 11px、斜體、`#9ca3af`：

- 對比度約 2.8:1，低於 WCAG AA 的 4.5:1
- 中文無真斜體，瀏覽器合成假斜體傷可讀性（排版界老共識）
- 11px 中文在手機上接近不可讀

而「條件：…」正是決定用戶拿不拿得到回饋的門檻資訊。另 `#9ca3af` 當文字色全站 24 處、
更淡的 `#d1d5db` 也有 5 處。

**方向**：條件文字 ≥13px、`#4b5563` 以上深度、取消斜體；提示性文字與必要條件視覺分級。

### 3. 訪客要按兩次「開始使用」才碰得到工具——入口雙重門檻【高（轉換率）】

首訪：landing 故事頁 → 按「開始使用」→ index 又一整頁產品介紹（三顆「開始使用」）→ 再按一次才見搜尋框。
比較工具的定式是「落地即可用」。

**方向**：帶 `?start` 進來的訪客直接顯示工具，intro 縮成工具上方一小條可收合介紹；完整故事留在 landing。

### 4. 排名結果在桌機用雙欄網格——排序閱讀順序模糊【中】

結果有名次意義（10% → 4.88% → 4%…）卻排成兩欄網格，需 Z 字掃描還原名次；
除第一名綠徽章外無名次標示。有序清單應單欄＋明確名次。

**方向**：桌機改單欄，或保留雙欄但加 1、2、3 名次角標。

### 5. 無標籤 icon 按鈕（mystery meat navigation）【中】

結果卡上的 ⓘ 與 📌 無文字或 tooltip，📌 功能無法猜測。右下角毛毛蟲浮動鈕（回報錯誤）
在手機上只剩圖示，且疊在結果卡／精選活動內容上方擋資訊。

**方向**：icon 補 tooltip＋`aria-label`；浮動回報鈕改小、加標籤、或移到不與內容重疊處。

### 6. 手機快捷支付 chips 硬截斷【中】

「LinePay／街口支付／Ap…」第三顆起被切一半，只有小 chevron 暗示還有更多，可發現性弱。

**方向**：橫向滑動並刻意露出「半顆」下一項，或換行＋「更多」展開。

### 7. 焦點指示局部被移除【低】

全站 16 處 `outline: none`，多數有 box-shadow 替代，但如 `.show-more-btn:focus` 只改文字色，
鍵盤使用者幾乎看不到焦點。

**方向**：統一 `:focus-visible` ＋明顯 focus ring。

### 8. 同名卡片多方案並列、差異埋在小字【低】

台新 Richart 卡、iLEO 信用卡各出現兩次（不同方案），唯一區別在條件小字（與發現 2 疊加更難辨）。

**方向**：卡名下加方案副標（如「台新 Richart 卡・Chill 刷方案」）。

### 9. 公告列＋精選活動雙輪播、6 秒自動輪換【低】

自動輪播用戶多半只看到第一則；現有 hover 暫停在觸控裝置無效，手機上等於不停輪換。

**方向**：touchstart 也觸發暫停，或公告改單則＋數字指示手動切換。

## 建議優先序

1、2 會讓用戶做出錯誤刷卡決策，優先處理；3 是轉換率問題；4–6 中等可用性改善；7–9 低成本收尾。
若動手，從 1（查無商家狀態列）與 2（條件文字樣式）開始——小改動、高回報、不碰回饋計算邏輯。
